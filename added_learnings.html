Self Study

Good interface design takes time and effort, but small, thoughtful decisions—such as considering real users and involving them early—can prevent serious design flaws. The Automatic syringe example shows how a numeric keypad interface led to a dangerous tenfold dosage error, a problem that was only discovered after consulting hospital staff.

Analyzing the Syringe Interface Using Different Models

The following sections explain how Norman’s Model, GOMS, and KLM each analyze the redesign of the syringe interface, which replaced a numeric keypad with increment and decrement buttons.

Norman’s Cycle of Action

Norman’s model highlights a large gulf of execution in the original keypad design. The user’s goal of setting an accurate dose was poorly supported, since a single extra key press could result in a fatal dosage. The redesigned button-based interface reduces this gulf by better matching the user’s intentions with safe, controlled actions.

GOMS Model

Using GOMS, the task of entering a dose is broken down into goals, methods, and operators. The keypad method allows faster input but provides limited support for detecting or correcting errors. In contrast, the button-based method is slower but forces users to verify each adjustment, improving safety during the task.

Keystroke-Level Model (KLM)

KLM focuses on predicting execution time for expert users. It would likely show that entering a dose with a keypad is faster due to fewer keystrokes, while button presses take longer. However, this comparison also demonstrates that speed alone is not an appropriate measure for safety-critical systems like medical devices.

The Importance of Iteration

Since people and contexts are complex, interface design rarely succeeds on the first attempt. Creating early prototypes and involving users throughout the design process helps identify serious flaws before they become permanently embedded in the system.

Norman’s Model of Interaction

Donald Norman is a key figure in Human–Computer Interaction and is well known for promoting user-centered design, usability, and cognitive engineering. His model of interaction is influential because it closely matches how users naturally think about and interact with computer systems.

According to Norman’s model, interaction begins when a user forms a goal and plans how to achieve it. The user then performs actions through the interface. After executing these actions, the user observes the system’s response and evaluates whether the outcome matches their original goal, deciding what to do next.

The Seven Stages of Action

Norman divides interaction into two main phases: execution and evaluation. Together, these phases consist of seven stages:

  1. Establishing the goal
  2. Forming the intention
  3. Specifying the action sequence
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the outcome against the goal

Gulfs of Execution and Evaluation

Norman uses this model to explain why users struggle with poorly designed interfaces. The gulf of execution refers to the gap between what a user wants to do and the actions the system makes available. The gulf of evaluation refers to the gap between what the system does and how easily the user can understand the result.

Human Error: Slips and Mistakes

Human errors can be divided into slips and mistakes. Slips occur when the user understands the system but performs the action incorrectly, such as mistyping or clicking the wrong button. Mistakes occur when the user forms the wrong goal due to misunderstanding the system, such as misinterpreting an icon’s meaning.

Slips can often be reduced through better interface design, like increasing spacing between buttons. Mistakes usually require deeper changes, such as redesigning the interface, improving training, or using clearer metaphors.

GOMS Model

The GOMS model, developed by Card, Moran, and Newell, describes how users complete tasks by breaking interaction into Goals, Operators, Methods, and Selection rules. It helps designers understand both the cognitive and physical steps involved in task performance.

Goals represent what the user wants to achieve. Operators are basic actions such as pressing a key or reading a dialog box. Methods describe different ways of achieving a goal, while selection rules determine which method a user chooses based on context or habit.

Design Focus: Why GOMS Saves Money

GOMS has been used in real-world system evaluations. For example, the telephone company NYNEX used GOMS to analyze a proposed computer system and discovered it would be slower than the existing one. As a result, the company abandoned the project, saving millions of dollars.

Keystroke-Level Model (KLM)

The Keystroke-Level Model is a detailed extension of GOMS that focuses on predicting execution time for expert users performing routine tasks. It is designed for short, well-defined tasks that usually take less than 20 seconds.

KLM assumes error-free performance and concentrates only on the execution phase of interaction. It breaks tasks into low-level operators such as key presses, mouse clicks, pointing, hand movement, mental preparation, and system response time.

By assigning standard time values to each operator and summing them, KLM allows designers to compare different interface designs or task methods and predict which one will be faster—without running full user experiments.

Lecture 1

HCI Overview

Human–Computer Interaction (HCI) combines concepts from Cognitive Psychology, Design, and Computer Science to understand how humans interact with technology.

Human Behavior and Memory

  • Humans often perform tasks based on how they were taught or what they observed at a young age (e.g., drawing a glass from memory).
  • Habits can be changed, but the fundamental nature of a person remains stable.
  • Humans store information in:
    • Short-Term Memory (STM)
    • Long-Term Memory (LTM)
    • Perceptual (Sensory) Storage

Interfaces and Models

  • Types of interfaces: GUI, CLI, voice-based, gesture-based
  • Cognitive Neural Networks (CNNs) resemble human processing by recognizing patterns such as texture, shape, and color.
  • Fitts’s Law predicts the time required to move to a target based on distance and size and influenced the invention of the mouse.
  • Chunking breaks large information into smaller units (e.g., 1234567890 → 123-456-7890).

UI vs UX

UI (User Interface) focuses on the visual design of a product and how it looks.

UX (User Experience) focuses on the overall experience and how the product feels to use, including usability and satisfaction.

The Automatic Syringe Case

The Automatic Syringe case demonstrates how small interface design decisions can lead to serious consequences. Numeric keypads caused dangerous input errors due to accidental key presses.

Interfaces rarely work perfectly on the first attempt, making iteration essential. Early prototypes should be easy to modify or discard, and rapid prototyping techniques such as paper sketches and mock-ups help designers test ideas quickly before full development.
